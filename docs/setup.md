## Как это работает

Приложение получает IP адрес и порт TURN relay из URL звонка VK (или берет эту информацию непосредственно из настроек). Звонок необходимо создать вручную и скопировать ссылку в настройки приложения (или найти ссылку через поиск).

Затем, с точки зрения iOS приложение строит туннель к TURN relay. С точки зрения самого TURN relay, приложение создает набор аллокаций (соединений) от клиентского устройства до него и указывает ему IP адрес и порт сервера, к которому уже строит аллокации (соединения) TURN relay. 

Чтобы принять эти соединения на сервере должно работать vk-turn-proxy приложение, которое реализует тот же протокол, что и клиентское приложение. В зависимости от выбранного режима работы сетевые пакеты оборачиваются различными дополнительными уровнями шифрования и имитации сетевых протоколов (но во всех режимах работы присутствует WireGuard слой) и отправляются к TURN relay поверх созданных соединений, который перекидывает их на сервер. 

Сервер снимает эти дополнительные уровни (кроме WireGuard) и отдает пакеты WireGuard серверу, который в общем случае должен быть запущен рядом на том же сервере. В клиентском приложении WireGuard часть встроена внутри, дополнительно запускать WireGuard на клиентском устройстве не надо. 

Обратное движение пакетов аналогичное - на сервере WireGuard отдает пакеты серверному приложению vk-turn-proxy, оно оборачивает пакеты дополнительными уровнями и отправляет TURN relay, тот отправляет клиентскому приложению.

## Что необходимо для работы

URL VK звонка (или адрес/порт TURN relay)

Выделенный или VPS сервер, на котором будет работать серверное приложение vk-turn-proxy. К нему будет подключаться TURN relay, поэтому сервер должен иметь публичный IP адрес, достижимый из Интернет и на firewall (если он есть) должен быть открыт доступ к паре IP:порт.

На сервере должен быть установлен, настроен и запущен [WireGuard сервер](https://www.wireguard.com)  (или какая-то другая реализация WireGuard протокола).

Клиентское приложение поддерживает 4 режима работы:

### Legacy

режим совместимости с [оригинальным](https://github.com/cacggghp/vk-turn-proxy) сервером. В настоящее время хотя и продолжает работать, но дает очень низкую скорость из-за шейпинга на стороне VK (если используется VK TURN relay).

### SRTP

режим по умолчанию для данного iOS приложения. Требует использования сервера, который понимает данный протокол. Доступно [здесь](https://github.com/anton48/vk-turn-proxy/releases). Необходимо скачать собранный бинарный файл для операционной системы, на которой работает ваш сервер и запускать там с ключом -srtp, например, так: 

/usr/local/bin/vk-turn-proxy-server -listen XXX.XXX.XXX.XXX:56000 -connect 127.0.0.1:51820 -srtp -logfile /var/log/srtp.server.log

здесь: 

XXX.XXX.XXX.XXX - публичный IP адрес сервера

56000 - порт, на котором будет слушать серверное приложение 

127.0.0.1:51820 - адрес и порт, на котором должен слушать WireGuard сервер

### SRTP-WRAP

режим совместимости с серверами других авторов:

https://github.com/Moroka8/vk-turn-proxy/releases

https://github.com/samosvalishe/vk-turn-proxy/releases

### SRTP-WRAP-A

режим совместимости с сервером [WDTT](https://github.com/amurcanov/proxy-turn-vk-android/releases). В этом сервере WireGuard встроен внутрь серверного приложения, поэтому отдельно запускать и настраивать его на сервере не надо. 

## Настройки клиентского приложения

### Legacy или SRTP режим

В разделе VK TURN Proxy необходимо ввести:

- URL VK звонка
- IP адрес и порт сервера, где запущена серверная часть vk-turn-proxy
- опционально адрес и порт непосредственно TURN relay
- выбрать режим подключения (server mode)
- опционально включить UDP в качестве транспорта (может быть несколько быстрее, чем TCP по умолчанию, но TCP значительно устойчивее)
- можно изменить количество соединений (10-20-30), от этого параметра зависит общая скорость работы с Интернет (на 30 соедиениях в идеальных условиях можно получить 50+ Мбит/сек)
- можно изменить таймаут при неудачном решении капчи:
<img width="750" height="914" alt="image" src="https://github.com/user-attachments/assets/0d81a804-170d-47cc-aece-97429ab758b1" />

В разделе WreGuard необходимо ввести:

- приватный ключ клиента (должен соответствовать публичному ключу в одном из разделов [Peer] на стороне WireGuard сервера)
- публичный ключ WireGuard сервера
- опционально preshared key (должен соответствовать PresharedKey в одном из разделов [Peer] на стороне WireGuard сервера)

<img width="750" height="619" alt="image" src="https://github.com/user-attachments/assets/ad317802-cdaf-4c46-9cc6-614067c74454" />

- IP адрес и маску (должны соответствовать AllowedIPs в одном из разделов [Peer] на стороне WireGuard сервера)
- IP адрес DNS сервера, который будет использоваться поверх туннеля
- IP сеть и маску для отправки в туннель (в общем случае должно быть 0.0.0.0/0, так как в iOS выбран режим includeAllNetworks=true)
